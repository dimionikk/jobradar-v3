# jobradar

Агрегатор вакансій для українського IT-ринку з AI-підбором та автоматизованим збором даних з декількох джерел.

**Продакшн:** [app.89-167-93-204.nip.io](https://app.89-167-93-204.nip.io)
**API документація:** [89-167-93-204.nip.io/docs](https://89-167-93-204.nip.io/docs)

## Функціонал

- Реєстрація та авторизація через JWT з blacklist-механізмом виходу через Redis
- Автоматичний збір вакансій з Djinni, DOU, Remotive, Work.ua кожні 6 годин
- Пошук та фільтрація вакансій за назвою, містом, джерелом
- Збереження вакансій та відстеження заявок зі зміною статусу (подано → співбесіда → офер/відмова)
- AI-підбір вакансій під профіль кандидата з поясненням оцінки відповідності
- Генерація персоналізованих супровідних листів через Anthropic API
- Адаптивний інтерфейс з окремою навігацією для мобільних пристроїв

## Стек

**Бекенд:** FastAPI, PostgreSQL, SQLAlchemy 2.0 (async), Alembic, Redis, Argon2, JWT, APScheduler, Anthropic API

**Фронтенд:** React, Vite, React Router, Tailwind CSS 4

**Інфраструктура:** Docker Compose, Nginx, Let's Encrypt, GitHub Actions CI/CD, Hetzner

```
├── components/    перевикористовувані UI-компоненти
├── pages/         сторінки застосунку
└── routes.js      централізовані маршрути

```

## Запуск локально

Потрібні Docker/Podman та `.env` файл (приклад — `.env.example`).

```bash
docker compose up -d --build
```

Бекенд буде доступний на `localhost:8000`, документація — на `localhost:8000/docs`.

Для фронтенду:

```bash
cd frontend
npm install
npm run dev
```

Доступний на `localhost:5173`.

## Тести

36 інтеграційних тестів на pytest, що покривають авторизацію, профіль, вакансії, збереження та заявки, включно з перевіркою ізоляції даних між користувачами.

```bash
pytest tests/ -v
```

Тести виконуються проти окремої тестової бази даних (`jobradar-test-db`), не торкаючись даних розробки.

## Деплой

CI/CD через GitHub Actions — кожен пуш у `main` автоматично деплоїть бекенд і фронтенд на сервер через `docker compose up -d --build`.
